# 📑 B站收藏夹跨端迁移工具 - 开发规划书 (极简直推版)

> **致执行开发的 Coding Agent：**
> 本次任务是使用 Python 开发一个后台守护工具。所有 B站相关 API 的调用逻辑、参数说明，**必须且只能以开源项目 `BACNext` (bilibili API Collect Next) 的官方 Apifox 文档 (https://github.com/BACNext/BACNext 为唯一依据)**。
> 
> **⚠️ 核心架构与业务约束（最高优先级）**：
> 本项目追求极致的“管道式”单向转移。**严禁引入任何本地数据库，严禁编写任何形式的「去重、查重、状态比对」逻辑（包括内存 Set 或预先拉取目标列表）。** 业务逻辑仅需做到：从源头拉取 -> 直接向目标发起添加请求。如果 B站 API 抛出“该视频已存在”之类的错误码，直接 Catch 并标记为成功/跳过即可，绝对不要为此增加前置查询判断。
> 项目使用已经创建好了的bilibili-videos-remove文件下的虚拟环境,禁止全局安装依赖。项目完成后生成部署指南

## 一、 技术栈约定

* **核心语言**：Python 3.10+ (需使用 `async/await` 异步机制)
* **网络请求**：`httpx` (使用 `AsyncClient` 降低高并发 IO 阻塞)
* **任务调度**：`APScheduler` (利用 `AsyncIOScheduler` 挂起定时检查与迁移任务)
* **扫码依赖**：`qrcode` (用于终端打印 ASCII 二维码供用户扫描)
* **日志记录**：`loguru` (负责轮转写入和控制台多级输出)

## 二、 核心模块拆解与实现要求

### 模块 1：双端鉴权与 Cookie 保活中心 (Auth & Cookie)
负责主、副两个账号的扫码登录与长效保活。

* **扫码登录（初始化）**：
    * 依据 BACNext 文档，调用【申请二维码】与【检查二维码状态】接口。
    * 在终端打印二维码，分别获取主账号和副账号的初始化 Cookie（需包含 `SESSDATA`, `bili_jct`, `DedeUserID`）以及 `refresh_token`。
* **Cookie 自动刷新机制**：
    * 将双端鉴权信息持久化写入本地 `config.json`。
    * 定期调用【检查是否需要刷新】接口。
    * 触发刷新时，严格按照 BACNext 记录的「Cookie 刷新流程」（含 RSA 公钥获取与加密环节）完成续期，并**立刻覆写 `config.json`**。

### 模块 2：纯粹单向转移引擎 (Direct Transfer Engine)
没有任何前置校验，直接一页一页搬运。

* **数据拉取（副账号源）**：
    * 调用读取收藏夹明细接口，携带副账号 Cookie 和源 `media_id`。
    * 实现分页提取 (`pn`, `ps`) 逻辑，按页获取源收藏夹内的所有视频 `bvid`。
* **数据写入（主账号目标）**：
    * 拿到源视频的 `bvid` 后，**直接、立刻**调用【操作收藏夹】接口，将视频加入主账号的 `add_media_ids` 中。
    * 必须正确携带主账号的 `csrf`（即 Cookie 中的 `bili_jct` 值）。
* **业务容错处理**：
    * 遇到由于“目标收藏夹已存在该视频”导致的 API 报错（具体表现为特定的业务 code），不抛出异常，直接通过 Loguru 打印一条 `INFO` 级别的“已存在，跳过”日志，并继续处理下一个。

### 模块 3：防风控护盾 (Anti-Ban System)
由于采取了无视状态的“全量直推”策略，必然会产生大量冗余的写请求（POST），因此**强制休眠策略是项目存活的唯一保障**。

* **伪装请求头**：全量注入固定版本的主流浏览器 `User-Agent`，携带合规的 `Referer` (`https://www.bilibili.com`) 和 `Origin`。
* **极度保守的强制操作延迟**：
    * **读操作（分页拉取）**：每页请求间强制随机休眠 **3~5 秒**。
    * **写操作（添加收藏）**：每次 POST 操作间必须强制随机休眠 **10~20 秒**。严禁使用并发任务(`asyncio.gather` 等) 批量发送添加请求。
* **风控熔断机制**：
    * 一旦接口返回 HTTP 403，或返回全局性的风控拦截码（如 `-412` 等），必须触发熔断：**立即挂起当前全部任务，强制休眠 4 小时以上**，打印 `ERROR` 报警，待休眠结束后自动重试。

### 模块 4：极简配置规范 (Configuration)
配置文件只需包含最核心的身份凭证与目标锚点。
* **`config.json` 示例结构**：
    ```json
    {
      "sub_account": { "cookie": "", "refresh_token": "", "source_media_id": "副账号源收藏夹ID" },
      "main_account": { "cookie": "", "refresh_token": "", "target_media_id": "主账号目标收藏夹ID" },
      "task_schedule": { "interval_hours": 24 }
    }
    ```
